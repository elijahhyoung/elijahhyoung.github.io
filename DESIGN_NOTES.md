# Design Notes

## 1. What is currently effective

- The existing site is simple, fast, and easy to deploy on GitHub Pages.
- It already identifies Elijah as an MIT builder working across robotics, manufacturing, research, and propulsion hardware.
- The headshot, resume PDF, yo-yo photography, ion thruster media, and card-shuffling poster provide useful source material.

## 2. What feels generic or weak

- The previous site read like a compact resume page rather than a technical portfolio.
- Projects were presented as similar cards, so the strongest hardware work did not feel central.
- The page did not expose constraints, methods, outcomes, media, or case-study depth.
- The visual system lacked an engineering identity beyond a clean card layout.
- Several project stories were missing media, build notes, or detailed technical decisions.

## 3. New visual concept

Engineering field notebook meets high-end hardware product launch: restrained MIT cardinal, black, white, and industrial gray; large condensed typography; thin rules; grid structure; measurement-like metadata; and authentic project media. The design emphasizes evidence, not decoration.

## 4. New content hierarchy

1. Hero: design-build-test positioning, MIT context, headshot, resume, contact/profile links.
2. Selected work: two large featured projects followed by focused research and propulsion cards.
3. Project case studies: static, shareable pages with challenge, constraints, approach, fabrication, testing, results, and next improvements.
4. Experience: concise industry/MIT overview without inventing resume details.
5. Capabilities: organized by how Elijah works - design, build, test, compute.
6. About: short technical narrative.
7. Contact: direct final prompt with email, resume, GitHub, and LinkedIn.

## 5. Missing media or information that would materially improve the site

- 2.007 robot photos, CAD, match video, mechanism details, and post-competition reflection.
- Yo-yo tooling/CAD screenshots, process parameters, material choice, defect/iteration notes, and final measurements.
- Card-shuffling exported plots, CV setup photos, and code/report links if public.
- Ion thruster labeled CAD sections, test stand diagram, instrumentation details, and measured test data.
- Structured resume text for TSMC and Burns & McDonnell responsibilities, dates, tools, and results.
